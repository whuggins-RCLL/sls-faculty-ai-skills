# Visuals and analogies

Use stack diagrams, data-flow maps, dependency maps, differentiation stacks, comparison matrices, action-authority ladders, claim-evidence tables, safe-trial flows, and decision trees. Every visual needs a text equivalent.

Analogy Card example: a domain application can be like a specialized cockpit around an engine supplied by another manufacturer. It explains workflow/control value over a third-party model. It breaks because AI products combine mutable software services, not fixed physical parts.
