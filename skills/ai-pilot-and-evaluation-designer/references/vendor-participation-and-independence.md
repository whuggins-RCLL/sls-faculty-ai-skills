# Vendor participation and independence

Vendors can help a pilot succeed and can also, intentionally or not, tilt it. Accept help while preserving independent judgment.

## When vendors assist

- Document their role in the pilot record.
- Preserve independent task and case selection; the vendor must not choose the entire test set.
- Preserve failed cases; do not let them be removed or reframed.
- Record every prompt, configuration, and setting change the vendor makes.
- Separate vendor support (setup, training) from evaluation judgment (scoring, conclusions).
- Do not let vendor scoring be the only scoring.
- Do not allow silent optimization between test rounds without recording it.
- Request exportable logs when appropriate.

## Independence checklist

| Question | Answer |
|---|---|
| Who selected the test cases? | |
| Who scored the outputs? | |
| Were failed cases preserved? | |
| Were configuration changes recorded? | |
| Was the baseline tuned as carefully as the AI workflow? | |
| Can results be reproduced without the vendor present? | |

## Balance

Do not assume vendor participation invalidates a pilot; assistance is normal. The goal is a record clear enough that a decision-maker can see what the vendor influenced and judge accordingly.

## Demo-to-pilot brief

When a pilot follows a vendor demonstration, produce a short brief that converts the demo's impressive moments into testable cases, adds the difficult and boundary cases the demo omitted, and defines who will score them.
