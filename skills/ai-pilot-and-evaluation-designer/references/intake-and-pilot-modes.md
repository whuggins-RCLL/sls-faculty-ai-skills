# Intake and pilot modes

## Conversational intake

Begin with one or two questions, not a form. Prioritize:

- What decision should this pilot help you make?
- What problem and workflow are being tested?
- What happens now, without the proposed AI use?
- What system and configuration are being considered?

Listen for the state the faculty member is actually in:

- A vendor demonstration they found impressive
- A five-minute experiment that worked
- A prototype or configured assistant
- A claim (accuracy, time saved, quality)
- A proposed teaching or research workflow
- An agent idea
- A candidate product to compare against an alternative

Reflect it back in one sentence, then propose the smallest useful mode.

## Modes

| Mode | Use when | Smallest useful output |
|---|---|---|
| Pilot-readiness check | Purpose or scope still fuzzy | Readiness Brief with findings and next step |
| Rapid pilot | Low stakes, quick signal needed | 8–15 case mini-protocol with thresholds |
| Full pilot protocol | Consequential decision | Complete protocol document |
| Test-set builder | Cases are the gap | Representative test-case set |
| Scoring and rubric builder | Review is inconsistent | Anchored rubric |
| Human-review study | "A human will check it" is the safeguard | Review-burden report design |
| Comparative pilot | Choosing among options | Comparison matrix design |
| Pilot rescue | A pilot is underway but weak | Diagnosis and repair plan |
| Results interpretation | Data exists, meaning unclear | Results report against predetermined criteria |
| Decision-memo builder | Pilot finished | Labeled decision memo |

## Sequencing rule

Never design a full protocol before a readiness check confirms the decision, baseline, and system identity are defined. If they are not, run readiness first and route missing problem framing to the Problem-to-AI Use Case Studio.

## One or two questions at a time

Keep the conversation moving in small steps. Summarize what is now defined before asking the next question. Offer to draft any output at any point rather than requiring the full protocol.
