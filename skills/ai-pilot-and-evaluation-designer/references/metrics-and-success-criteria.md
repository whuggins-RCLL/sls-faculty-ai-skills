# Metrics and success criteria

Select metrics from the decision, not from what is easy to count. Define thresholds before seeing results.

## Metric dimensions

Accuracy; source existence; proposition support; citation completeness; attribution; currency; material completeness; omission; reliability; variability; relevance; usefulness; preparation time; prompting time; verification time; correction time; escalation time; reviewer burden; error severity; detectability; reversibility; data exposure; permission overreach; accessibility; equity; integration effort; exportability; logging; support burden.

Pick the few that would actually change the decision. Time saved is meaningless if verification time erases it, so pair any efficiency metric with a quality and a review-burden metric.

For qualitative measures, use anchored descriptions rather than numbers (for example, "no fabricated authorities; every proposition traceable to a real, on-point source").

## Do not

- Do not invent numbers merely to appear scientific.
- Do not reduce materially different consequences to one aggregate score. A fabricated citation and a formatting slip are not fungible.
- Do not change thresholds after seeing results. If a threshold was wrong, say so explicitly and rerun.

## Comparative evaluation matrix

For option comparisons, hold the test set and scoring constant and record per option:

| Option (exact config) | Quality | Critical failures | Review burden | Data exposure | Accessibility | Cost | Notes |
|---|---|---|---|---|---|---|---|

A single blended score hides the tradeoffs a decision-maker needs to see; keep the columns separate.
