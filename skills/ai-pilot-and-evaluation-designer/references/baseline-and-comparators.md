# Baseline and comparators

Without a baseline, "the AI did well" has no meaning. Establish what "well" is measured against before testing.

## Baseline options

- Current faculty workflow
- Current staff-assisted workflow
- Manual review
- Existing template
- Existing search system
- Existing non-AI software
- Another authorized product
- General-purpose AI workflow
- Expert reference answer
- No intervention

Choose the baseline that matches the decision. If the choice is "adopt this product vs. keep doing what we do now," the baseline is the current workflow, not a blank page.

## Record the current state

When available, capture for the baseline:

- Current time per task
- Current quality
- Current errors
- Current omissions
- Current user burden
- Current review effort
- Current cost
- Current access and accessibility
- Current failure handling

## Do not invent baseline measurements

If current numbers are unknown, say so and either measure a small sample or use qualitative anchored descriptions. Never fabricate a baseline figure to produce a clean comparison.

## Fair comparison rules

- Give the baseline and the AI workflow the same tasks and the same review standard.
- Include the human effort required by each (preparation, prompting, verification, correction).
- Include a non-AI comparison when one is realistic.
- Do not let a vendor tune the AI workflow while the baseline is left un-tuned.

## Comparative pilots

When choosing among options, define each comparator's exact configuration and hold the test set and scoring constant. See the comparison matrix in the metrics reference and the decision memo template.
