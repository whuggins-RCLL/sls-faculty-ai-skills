# AI Policy Patterns

Provide both a syllabus-ready section and a separate editable policy document with faculty-facing implementation notes.

## Restricted pattern

State the pedagogical reason, permitted support tools if any, prohibited generated work, rules for research leads and private study, disclosure expectations, exam/confidential-material rules, and how to ask about uncertainty.

## Mixed pattern

Use assignment-specific tables: permitted uses, prohibited uses, prior-permission uses, required verification, disclosure/acknowledgment, prompt/output retention, and confidentiality constraints.

## Infused pattern

Connect AI activities to course objectives. Include expectations for verification, critique, reflection, disclosure, professional responsibility, bias/confidentiality analysis, and human accountability.

## No separate policy

If intentionally selected, state that no separate AI policy is included and identify which existing course or institutional rules govern student work, using placeholders for official language.
