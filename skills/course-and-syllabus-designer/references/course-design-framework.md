# Course Design Framework

## Course design brief fields

Include title, type, intended students, prerequisites, central questions, purpose, teaching philosophy, constraints, AI posture, materials strategy, assumptions, and unresolved faculty decisions.

## Architecture

Organize units/modules by conceptual progression. For each unit identify purpose, dependencies, estimated class time, source categories/readings, assessments, and learning-objective connections.

## Calendar

For each session include number, date if supplied, topic, central question, assigned material, preparation, teaching method, objective, assessment/follow-up, and notes/unresolved items.

## Faculty voice

Offer alternatives and rationales. Do not flatten distinctive pedagogy into generic best practices. Preserve elements faculty marks as non-negotiable.
