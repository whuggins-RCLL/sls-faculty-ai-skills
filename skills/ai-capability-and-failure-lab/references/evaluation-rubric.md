# Evaluation rubric

## Activation

Activate for behavior testing, failure-mode experience, output diagnosis, classroom lab design, comparison labs, and capability profiles.

Do not activate for: generic LLM explanation, full product due diligence, broad claim cross-examination, complete pilot design, course policy drafting, syllabus creation, substantive legal research, student misconduct adjudication.

## Experimental integrity checks

- [ ] One main variable isolated
- [ ] Prediction before result
- [ ] Confidence recorded (Low / Moderate / High)
- [ ] Reference answer or review method defined
- [ ] Multiple trials when variability is the question
- [ ] Conditions documented
- [ ] Safe materials default
- [ ] What result does not prove stated
- [ ] Failure layer not assumed prematurely
- [ ] Follow-up test proposed when useful

## Safety checks

- [ ] Fictional authority labeled
- [ ] No deceptive unlabeled fake citations
- [ ] No harmful security or injection testing
- [ ] No confidential data requested casually
- [ ] No student misconduct adjudication
- [ ] No hidden chain-of-thought requested

## Distinction checks

- [ ] Capability vs reliability
- [ ] Success vs deployment evidence
- [ ] Failure vs universal uselessness
- [ ] Source existence vs source support
- [ ] Retrieval vs interpretation
- [ ] Model vs product vs workflow failure
- [ ] Fluency vs accuracy
- [ ] One trial vs general conclusion

## Tone checks

- [ ] Does not shame incorrect predictions
- [ ] Does not frame failures as dishonesty
- [ ] Does not celebrate gotchas
- [ ] Does not overpraise success
- [ ] Does not overwhelm with methodology
- [ ] Does not imply nontechnical users cannot evaluate AI
- [ ] Does not force confident conclusion

## Legal material checks

- [ ] Real authority verified when used
- [ ] Citation support checked
- [ ] Jurisdiction and date considered
- [ ] Omission and procedural posture tested
- [ ] No citator claim without citator

## Handoff checks

- Product due diligence → Product Decoder
- Broad claims → Claims Cross-Examiner
- System layers → System Anatomy Lab
- Use-case fit → planned Use Case Studio
- Multi-case evaluation → planned Pilot Designer

## Web research checks

- Current product facts researched when named
- Research date recorded
- Limits stated when web unavailable
