# Experiential learning cycle

## Predict → Test → Inspect → Compare → Explain → Revise → Apply

Every substantial lab should make the cycle visible, not merely mention it.

## Predict

Before running or revealing results, ask:

- What do you predict will happen?
- How confident are you? (Low, Moderate, High)
- What evidence or assumption drives that prediction?

Prediction is not a graded quiz. Incorrect predictions that improve mental models are valuable.

## Test

Run the smallest experiment that can reveal one behavior. Document conditions before execution.

## Inspect

Examine accuracy, source support, omission, consistency, relevance, uncertainty, instruction following, review burden, and error severity without immediately explaining away surprises.

## Compare

Compare against authoritative answers, supplied sources, non-AI baselines, alternate prompts, additional trials, or different systems when actually available.

## Explain

State what the result suggests under these conditions. Separate observation from interpretation.

## Revise

Ask what changed in understanding or confidence. Distinguish:

- Correct prediction with sound reasoning
- Correct prediction for a weak reason
- Incorrect prediction that improves the mental model
- Result too ambiguous to update confidence
- One trial that should not change confidence much

## Apply

Connect to teaching, research, product review, or institutional use. Propose a follow-up experiment when warranted.

## Faculty-facing cycle card

| Step | Faculty question |
|------|------------------|
| Question | What behavior are we investigating? |
| Prediction | What do I expect? |
| Confidence | How sure am I? |
| Test | What is the smallest fair test? |
| Observe | What actually happened? |
| Inspect | What does the evidence show? |
| Compare | What differs from reference or alternate? |
| Explain | What does this suggest? |
| Revise | What do I think now? |
| Apply | What will I do with this? |
