# Capability taxonomy

Capabilities are task possibilities under stated conditions—not guarantees of reliability.

For each capability, specify: task conditions, evidence of success, common failure modes, verification method, appropriate and inappropriate generalizations, potential faculty application.

## Categories

| Capability | Task conditions | Evidence of success | Common failures | Verification |
|------------|-----------------|---------------------|-----------------|--------------|
| Generating alternatives | Clear problem, bounded scope | Multiple distinct viable options | Repetition, hidden assumptions | Compare quality and diversity |
| Rewriting text | Source text supplied | Preserves meaning, meets style goal | Drift, omission, added claims | Side-by-side review |
| Summarizing supplied material | Complete source provided | Accurate, proportional summary | Omission, invention, overconfidence | Check against source |
| Extracting information | Structured or unstructured input | Correct fields or facts | Hallucinated fields, misread | Spot-check against source |
| Classifying items | Clear categories, examples | Consistent labels | Boundary errors, inconsistent rules | Confusion matrix on test set |
| Comparing supplied sources | Multiple sources provided | Fair comparison, tensions noted | False equivalence, silent omission | Source-support audit |
| Explaining concepts | Audience and depth specified | Accurate, accessible explanation | Oversimplification, false certainty | Expert review |
| Translating terminology | Domain specified | Consistent term mapping | False friends, lost nuance | Bilingual or expert check |
| Generating questions | Topic and purpose stated | Relevant, varied questions | Trivial, off-topic, biased | Faculty review |
| Creating hypotheticals | Doctrine and constraints given | Legally coherent variation | Wrong rule, missing posture | Expert review |
| Structuring information | Input materials supplied | Logical organization | Misordered priority, dropped items | Compare to reference outline |
| Retrieving source passages | Corpus or files available | Relevant passage selected | Missed passage, wrong doc | Check corpus directly |
| Using external tools | Tool actually available | Correct tool use, accurate report | False tool claim, wrong result | Inspect tool logs or repeat |
| Following multi-step instructions | Steps enumerated | Order and constraints respected | Skipped steps, substitution | Step checklist |
| Producing code | Spec and environment stated | Runs or meets spec | Subtle bugs, insecure patterns | Execute or test |
| Analyzing structured data | Dataset supplied | Correct calculations, patterns | Wrong column, invented stats | Recompute |
| Identifying patterns | Examples or data provided | Plausible pattern named | Overfit, spurious correlation | Holdout examples |
| Critiquing an argument | Argument text supplied | Identifies weaknesses fairly | Straw man, missed strongest point | Expert review |
| Generating counterarguments | Position and sources given | Relevant counterpoints | Weak counters, fabricated authority | Source check |
| Producing source-grounded responses | Authorized sources attached | Claims tied to sources | Citation mismatch, omission | Source-support audit |

## Appropriate generalizations

A capability demonstration under tested conditions may justify: "worth a narrow follow-up test" or "potentially useful with verification."

## Inappropriate generalizations

Do not infer: deployment readiness, legal accuracy, equity, security, cost savings, or transfer to all models and tasks.

## Profile language

Use: strong performance on this narrow task; useful with straightforward verification; promising but variable; useful for generating alternatives, not final answers.
