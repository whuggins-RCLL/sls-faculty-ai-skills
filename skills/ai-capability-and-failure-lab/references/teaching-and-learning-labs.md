# Teaching and learning labs

Student-facing AI-literacy exercises. Every exercise includes the components below.

## Required components

- Learning objective
- Required preparation
- AI access assumptions
- No-tool alternative
- Data restrictions
- Activity steps
- Evidence students must inspect
- Debrief questions
- Faculty notes
- Accessibility considerations
- Estimated time
- Assessment option (optional)
- Clear statement of what the exercise does not prove

## Example teaching labs

| Lab | Objective | No-tool alternative |
|-----|-----------|---------------------|
| Compare AI and human summaries | Evaluate completeness | Compare two human summaries |
| Verify citations | Source-support audit | Manual source check only |
| Identify missing authorities | Omission detection | Checklist on written analysis |
| Diagnose overbroad rules | Scope calibration | Doctrine mapping exercise |
| Compare framed prompts | Framing sensitivity | Rewrite question variants |
| Observe variability | Stability awareness | Compare two human drafts |
| Examine confidence language | Calibration | Mark confident claims on paper |
| Critique AI feedback | Evaluation skill | Peer critique |
| Drafting vs judgment | Workflow boundaries | Outline-only assignment |
| Automation bias | Trust calibration | Polished vs rough equal-error samples |
| Verification checklist design | Process skill | Build checklist without AI |
| One success proves little | Reliability reasoning | Discuss single-sample stats |
| Revise AI answer with sources | Grounding practice | Revise human draft with sources |

## Access and equity

Do not require students to create accounts without noting access and equity concerns. Offer no-tool alternatives.

## Fictional materials

Label all fictional legal content prominently for student exercises.

## Classroom exercise output format

Include faculty overview, student instructions, materials, no-tool option, timing, accessibility, data rules, debrief, optional assessment, faculty answer notes.
