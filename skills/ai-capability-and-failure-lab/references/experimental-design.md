# Experimental design

## Smallest useful experiment

Prefer a narrow experiment that reveals one behavior. Avoid testing several variables simultaneously.

## Lab record structure

Use when documenting a lab:

- Lab Question
- Prediction
- Confidence
- Test Setup
- Input
- Expected or Reference Result
- Trials
- Observations
- Evidence Check
- Provisional Finding
- What This Does Not Prove
- Revised Mental Model
- Faculty Application

## Variables and controls

Identify:

- **Independent variable** — what you change (prompt phrase, source set, framing, trial number).
- **Dependent variable** — what you observe (conclusion, citation, omission, confidence language).
- **Controls** — what stays the same (model, product, account tier, tools, materials).
- **Comparison** — authoritative answer, alternate prompt, second trial, human baseline.

Change one meaningful element at a time for prompt-sensitivity tests.

## Trials

Use multiple trials when variability, stability, or consistency is the question. One trial is sufficient for some capability demonstrations but not for reliability claims.

## Success and failure criteria

Define before running:

- **Success criteria** — what would count as performing the task adequately under this test?
- **Failure criteria** — what would count as a meaningful failure (fabrication, omission, unacknowledged gap, instruction conflict)?

## Conditions to document

- Model, product, feature, version when known
- Date tested
- Account tier
- Tools, retrieval, web search
- Files supplied
- Exact prompt or instruction
- Number of trials
- Review method for expected answer

## Limitations block

Every lab ends with explicit limits: what one trial, one product version, or one prompt cannot establish.

## Follow-up test

Propose the next narrow experiment when the result is ambiguous or when a layer attribution needs confirmation.
