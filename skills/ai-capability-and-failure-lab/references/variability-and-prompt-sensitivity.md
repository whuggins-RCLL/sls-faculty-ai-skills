# Variability and prompt sensitivity

## Variability lab

**Question:** Are materially identical requests stable?

**Procedure:**

1. Fix all conditions (product, model, tools, prompt, materials).
2. Run three or more trials.
3. Compare conclusion, sources, confidence, omissions, structure, recommendations.

**Variability comparison output:**

| Element | Trial 1 | Trial 2 | Trial 3 | Meaningful difference? |

**Interpretation:**

- Stylistic variation alone may not indicate unreliability.
- Different legal conclusions, missing sources, or changed risk assessments usually matter.

**Does not prove:** Universal instability across all tasks and versions.

## Prompt-sensitivity lab

**Question:** Does one meaningful wording change alter the result?

**Procedure:**

1. Start with baseline prompt.
2. Change one element: role, audience, jurisdiction, outcome preference (neutral test), output format, or scope.
3. Compare outputs.
4. Label which differences are justified.

**Avoid:** Comparing unrelated requests and calling the difference prompt sensitivity.

## Prompt-difference map

Document:

- Baseline prompt
- Changed element
- Observed difference
- Justified? (yes/no/unclear)

## Connection to reliability

Variability and prompt sensitivity inform reliability under specific conditions—they do not by themselves define deployment readiness.
