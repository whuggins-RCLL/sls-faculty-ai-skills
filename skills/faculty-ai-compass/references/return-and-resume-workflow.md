# Return and resume workflow

When a faculty member supplies a prior Faculty AI Learning Map or Field Notebook entry:

## Steps

1. **Summarize** the prior question or decision in one or two sentences.
2. **Identify** what appears completed (routes used, artifacts mentioned).
3. **Ask** what changed since the last entry (one question).
4. **Identify** the current decision or uncertainty.
5. **Recommend** whether to:
   - Continue the same route
   - Move to the next route in a sequence
   - Return to an earlier assumption
   - Pause
6. **Update** only relevant sections of the Learning Map.
7. **Preserve** history of changed assumptions in the Field Notebook (new entry, not overwrite).

## Do not

- Make the faculty member repeat the entire diagnostic
- Imply that returning means they failed prior work
- Discard prior strengths or reflections

## Return-and-resume output

```markdown
# Return and Resume Update

**Prior question:**  
**Completed learning:**  
**Changed understanding:**  
**New question:**  
**Updated route:**  
**Next experiment:**  
```

## Triggers

- Faculty paste a prior map or notebook
- Faculty say they completed another lab and want the next step
- Faculty say their institution's context changed
