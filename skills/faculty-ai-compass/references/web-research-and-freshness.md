# Web research and freshness

## Trigger

Research when current information matters for routing or learning-map notes:

- Named products, models, providers
- Stanford services and public listings
- Pricing, tiers, features
- Security and data-use terms
- Benchmarks and headlines
- Institutional guidance pages

## Protocol

1. Use official Stanford sources first for Stanford claims
2. Use primary vendor or provider sources for product facts
3. Distinguish confirmed, vendor-claimed, inferred, and unknown
4. Include research checked date in outputs when web research is used

## Freshness note (when applicable)

```markdown
**Current guidance or product facts requiring verification:**
- [Item] — checked [date]; local confirmation may still be required
```

## No web access

- Provide stable conceptual routing
- Do not make current product or institutional claims from memory
- Mark items that require verification when connectivity returns

## Evaluations note

Regression tests should verify routing behavior, not frozen current product facts.
