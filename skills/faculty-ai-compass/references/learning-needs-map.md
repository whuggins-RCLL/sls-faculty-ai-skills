# Learning needs map

A learning needs map is an internal working structure before producing a Faculty AI Learning Map. It is not shown to faculty as a scorecard.

## Dimensions (select as needed)

| Dimension | Signals | Likely route |
|---|---|---|
| System mental model | Confuses model, product, retrieval, agent, training | System Anatomy Lab |
| Product judgment | Named product, demo prep, vendor terms | Product Decoder |
| Evidence judgment | Headline, benchmark, productivity claim | Claims Cross-Examiner |
| Problem framing | Tool-first idea, undefined problem | Problem-to-AI Use Case Studio |
| Direct experience | Wants to see failure modes | Capability and Failure Lab |
| Evaluation readiness | Defined pilot, baseline, reviewers | Pilot and Evaluation Designer |
| Governance | Permissions, review, stopping authority | Governance Simulator |
| Teaching transfer | Course, assignment, student learning | Teaching Transfer Studio |

## Existing strengths to note specifically

Examples (use only when demonstrated):

- Recognizes that a model and a product differ
- Asks about evidence rather than accepting a demonstration
- Notices data or permission concerns
- Separates capability from appropriate use
- Understands that citations require verification
- Identifies the need for a non-AI comparison
- Preserves faculty or professional judgment
- Recognizes that learning objectives should precede tools
- Expresses uncertainty accurately
- Knows where institutional confirmation is needed

## Uncertainty framing

Frame gaps as questions, not deficits:

- "What remains unclear is whether retrieval covers the full source set."
- "The open question is who can stop the workflow if the agent misroutes a file."

## Output

Translate the learning needs map into `faculty-ai-learning-map-template.md` when the faculty member wants a saved route.
