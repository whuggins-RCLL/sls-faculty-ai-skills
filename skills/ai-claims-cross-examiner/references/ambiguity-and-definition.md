# Ambiguity and definition

Define ambiguous terms before debating conclusions. For each important term:

1. Identify likely meanings.
2. Explain which meaning the available evidence supports.
3. Avoid treating marketing terms as technical standards.
4. State when the claim changes depending on definition.

## High-priority ambiguous terms

AI; reasoning; understanding; intelligence; accuracy; hallucination; reliable; safe; secure; private; approved; autonomous; agent; proprietary; open source; open-weight; trained on; learns from; grounded; verified; expert-level; human-level; better than lawyers; faster; efficient; transformative; bias; explainable; transparent; compliant.

## Definition workflow

1. Quote the term in context.
2. List two or three plausible definitions (technical, marketing, colloquial).
3. Identify which definition the speaker likely intends.
4. State which definition the evidence actually addresses.
5. Note if the claim is definition-dependent.

## Provisional label

Use **Definition-dependent** when the verdict changes materially based on how a key term is defined.

## Plain-language rule

Explain definitions without requiring statistical or computer-science jargon. Use operational definitions: what would you observe if the term were true?

Example for “reasoning”: producing multi-step answers; following logical rules on a test; or human-like understanding. Each implies different evidence.
