# Intake and Routing

Begin from the faculty member's material: course title, topic, date, duration, level, enrollment, prior knowledge, syllabus section, readings, cases, statutes, articles, slides, notes, prior plans, questions, assessment plans, teaching preferences, past problems, accessibility or modality requirements, AI posture, and real or fictional material preferences.

Ask only essential questions. If the user says, for example, "I need to teach a 90-minute class on personal jurisdiction," proceed with assumptions and ask at most two high-impact questions, such as course level and assigned materials.

## Paths

- Path A: Build a new class session when the user provides a topic or goal but no complete plan.
- Path B: Revise an existing session when the user supplies a lesson plan, notes, slides, or outline. First identify what to preserve, what did not work, timing problems, weak transitions, redundant questions, objective misalignment, preparation issues, accessibility concerns, formative-feedback opportunities, and whether the session is too ambitious.
- Path C: Create a targeted component when the user asks for only an opening, Socratic sequence, hypothetical, simulation, debrief, exit ticket, timing revision, board plan, slide outline, remote adaptation, AI activity, or reading guide.
- Path D: Create a connected sequence for two to four meetings when the topic cannot reasonably fit in one class. Show progression and avoid duplication.

## Connectors

Use Google Drive or calendar only if connected and authorized. If unavailable, ask the faculty member to upload or paste relevant text. Do not require connector access.
