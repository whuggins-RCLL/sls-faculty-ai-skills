# Governance

This repository is governed as faculty-centered instructional infrastructure. AI skills should support, not replace, educator judgment.

## Roles

- **Faculty owners:** Approve pedagogical goals, course policies, assessment designs, and final instructional materials.
- **Maintainers:** Review repository structure, skill quality, evaluation coverage, and version control hygiene.
- **Policy reviewers:** Validate alignment with institutional, accessibility, privacy, copyright, and academic integrity requirements.
- **Learner advocates:** Identify potential equity, accessibility, clarity, and student experience concerns.

## Decision principles

1. Faculty academic judgment takes precedence over generated suggestions.
2. Institutional policy requirements must be explicit and locally verified.
3. Skills should document uncertainty and preserve source context.
4. Evaluations should cover realistic, high-impact use cases before broad rollout.
5. Changes that affect privacy, grading, accessibility, or academic integrity require elevated review.

## Change management

- Minor copy edits may be reviewed by maintainers.
- New skills or substantial workflow changes should receive faculty-owner review.
- Policy-sensitive changes should receive policy-reviewer approval.
- Evaluation gaps should be documented before release when immediate coverage is not feasible.
