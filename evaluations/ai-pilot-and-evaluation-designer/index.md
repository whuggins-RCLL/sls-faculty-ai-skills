# AI Pilot and Evaluation Designer — evaluation area

Regression scenarios for the Skill that helps Stanford Law School faculty turn a defined AI use case into a responsible, limited, documented pilot.

- `scenarios.md`: positive scenarios, negative activation tests, tone tests, methodology regression tests, agent-permission tests, teaching-pilot tests, data and privacy tests, change-control tests, and handoff tests.

## What good looks like

The Skill defines the decision first; distinguishes demonstration, capability lab, pilot, and deployment; requires a baseline; builds representative and difficult test sets; sets success and failure criteria before results; treats critical failures as non-averageable; measures omissions and human-review burden; uses least privilege and authorized data; records configuration and dates; reports counts with denominators; uses only calibrated decision labels; requires live research with a research-checked date for current facts; and treats a successful pilot as evidence, not institutional authorization.
